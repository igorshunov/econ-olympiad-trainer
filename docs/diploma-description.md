# Описание дипломного проекта

**Тема:** Адаптивный тренажёр для подготовки к олимпиадам по экономике
с индивидуальным образовательным треком на основе графа тем-пререквизитов

**Автор:** Горшунов Игорь Станиславович
**Группа:** 09.02.07 ОЗФ
**Год:** 2025-2026

---

## 1. Актуальность темы

Подготовка к олимпиадам школьного и студенческого уровней по экономике
(Всероссийская олимпиада школьников, олимпиада «Высшая Проба» НИУ ВШЭ,
МГУ им. М. В. Ломоносова, региональные олимпиады) на сегодняшний день
обеспечена большим количеством открытых архивов задач с разборами и
ответами. Однако каждый такой архив представляет собой неструктурированную
ленту: задачи перечислены либо по годам, либо по этапам, без учёта темы
и уровня сложности. Это создаёт три проблемы для участника:

- невозможно понять, **с чего начать** — пытаясь решать задачи подряд,
  ученик сталкивается одновременно с пробелами в базовых темах
  (например, эластичность спроса) и со сложными темами на верху иерархии
  (моделями общего равновесия), что демотивирует;
- невозможно сфокусироваться на **слабых местах** — нет инструмента
  диагностики, который определял бы, какие именно темы требуют доработки;
- существующие коммерческие сервисы подготовки (Фоксфорд, Stepik,
  отдельные курсы) построены на линейном «прохождении модулей», а не на
  адаптации к фактическому уровню конкретного ученика.

Адаптивные системы обучения (Knewton, ALEKS, Duolingo) показывают, что
**персонализированный подбор задач по графу пререквизитов** позволяет
сократить время подготовки в 1,5-2 раза по сравнению с линейным курсом.
Для русскоязычной подготовки к олимпиадам по экономике аналогичных
открытых решений на момент проектирования не существует.

## 2. Целевая аудитория

- **Старшеклассники (10-11 класс)**, готовящиеся к олимпиадам по
  экономике уровня регионального и заключительного этапов ВсОШ;
- **Студенты 1-2 курса** экономических, финансовых и IT-направлений,
  которые используют олимпиадную подготовку как способ углубить базу
  по микро- и макроэкономике;
- **Преподаватели и наставники**, ведущие кружки и факультативы — для
  них сервис даёт срез по группе («какие темы провалены большинством»);
- **Самостоятельно изучающие** экономику взрослые, которым нужен
  инструмент с обратной связью без покупки полноценного курса.

Ядро аудитории: 1 000-3 000 активных пользователей в течение учебного
года, с пиком в феврале (региональный этап) и марте-апреле (заключительный
этап и подготовка к ЕГЭ).

## 3. Предполагаемый функционал

### 3.1 Базовый сценарий

1. Пользователь регистрируется (e-mail + пароль) или входит как «гость»
   для пробной сессии.
2. На главном экране отображается **граф тем** — ориентированный граф,
   где узлы — темы (спрос-предложение, эластичность, поведение фирмы,
   рыночные структуры, ВВП, инфляция, ДКП и т.д.), а рёбра — отношения
   «является пререквизитом». Цветом узла показывается текущая оценка
   владения темой (mastery) — от красного к зелёному.
3. Пользователь нажимает «Следующая задача» — система предлагает задачу
   из темы с mastery около 0,5 (граница ближайшего развития) и
   подходящей сложностью.
4. После задачи пользователь оставляет один из четырёх ответов:
   - **«Решил»** — mastery текущей темы повышается;
   - **«Не решил»** — mastery понижается, следующая задача берётся из
     темы-пререквизита;
   - **«Слишком просто»** — mastery текущей темы повышается агрессивно,
     следующая задача берётся из downstream-темы;
   - **«Слишком сложно»** — mastery понижается, следующая задача берётся
     из upstream-темы с более низкой сложностью.
5. Все попытки фиксируются. Пользователь может посмотреть историю и
   текущий профиль владения темами.

### 3.2 Расширенный функционал

- Импорт задач из открытых источников (HTML/PDF архивы) в общую базу,
  с разметкой темы и сложности;
- Режим «контест» — ограниченный по времени набор задач без подсказок;
- Экспорт прогресса в PDF (для портфолио, отправки наставнику);
- Социальный слой: сравнение прогресса с группой / другом, не показывая
  конкретные ответы;
- Тёмная тема, мобильная адаптация, клавиатурные сокращения.

## 4. Используемые технологии (предварительный стек)

| Слой | Технология | Обоснование |
|------|------------|-------------|
| Frontend | HTML5, CSS3 (Flex/Grid), vanilla JavaScript (ES2022) | Минимум зависимостей для лёгкого деплоя на GitHub Pages |
| Визуализация графа | Cytoscape.js | Открытая библиотека, специально для графов знаний |
| Хранилище клиента | localStorage / IndexedDB | Демо-режим без бэкенда |
| Backend (на следующих этапах) | Python 3.11 + FastAPI | Быстрая разработка REST API, типизация |
| СУБД | PostgreSQL 16 | Реляционная модель с CTE и оконными функциями для аналитики; SQLite — для разработки |
| ORM | SQLAlchemy 2.x | Стандарт для FastAPI-проектов |
| Аутентификация | JWT + bcrypt | Безопасно и просто |
| Деплой фронта | GitHub Pages | Бесплатно, CI через Actions |
| Деплой бэка (план) | Render.com / Railway / Yandex Cloud | Free-tier для дипломной демонстрации |

В рамках учебной практики УП.02 реализуется **фронтенд-MVP** с
встроенными JSON-данными — этого достаточно для демонстрации основной
адаптивной логики. Бэкенд-часть с реальной СУБД проектируется и
прорабатывается в рамках УП.11.

## 5. Ожидаемый результат

К концу дипломного проекта планируется получить:

1. Веб-приложение, доступное по публичному URL, в котором пользователь
   может зарегистрироваться, увидеть граф тем по экономике, решать
   задачи с адаптивным подбором следующих, и наблюдать рост mastery по
   темам;
2. База данных в PostgreSQL с минимум 200 задачами, размеченными по
   ~30 темам, образующим осмысленный граф пререквизитов;
3. REST API для всех клиентских операций (логин, получение следующей
   задачи, фиксация ответа, чтение прогресса);
4. Документация: README, схема БД (ER-диаграмма), use-case-диаграмма,
   архитектурная схема, инструкция по локальному развёртыванию;
5. Демонстрационное видео работы сервиса (3-5 минут);
6. Возможный задел на дальнейшее развитие — расширение на смежные
   предметы (математика, физика) без изменения архитектуры, поскольку
   модель «задачи + темы как граф + mastery» предметно-нейтральна.

В качестве защитного итога — публичный репозиторий с открытой лицензией
(MIT), пригодный для использования преподавателями экономики при
подготовке учеников к олимпиадам.
